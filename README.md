# IT Jobs Statistics

Веб-сервис анализа рынка IT-вакансий для дипломного MVP. Приложение помогает соискателю понять, в каком направлении развиваться: где больше вакансий, насколько высокая конкуренция и какие навыки чаще всего требуют работодатели.

## Что реализовано

- Flask-приложение с русскоязычным dashboard.
- PostgreSQL как основной источник данных для локального demo-режима.
- Фильтры по специальности, грейду, навыку и формату работы.
- Метрики: количество вакансий, количество активных резюме, коэффициент конкурентности.
- Графики Chart.js: грейды, требуемый опыт, формат работы.
- Таблица топ навыков с количеством вакансий и долей.
- Автоматические выводы: оценка конкуренции, главный навык, основной грейд спроса.
- Fallback для направлений или фильтров без вакансий.
- Тесты для маршрута, конфигурации и analytics-контракта.

## Архитектура

```text
[Browser]
    ↓
[Flask + Jinja2 + Chart.js]
    ↓
[PostgreSQL]
    ↑
[Parser / import layer]
```

Ключевой контракт находится в `app.analytics.get_dashboard_data`. UI зависит от этого контракта, а не от конкретного источника данных. Сейчас данные берутся из PostgreSQL, а mock-провайдер остается fallback на случай недоступной БД.

## Данные

Для защиты используется подготовленная локальная выборка:

- вакансии по Backend Python, Frontend, Data Analyst и DevOps;
- агрегированные резюме по специальностям и грейдам;
- `QA Engineer` как пример направления без вакансий;
- персональные данные резюме не хранятся.

Файл схемы: `docker/postgres/init/001_schema.sql`.
Файл demo-данных: `docker/postgres/init/002_seed.sql`.

## Локальный запуск

```bash
docker compose up
```

После запуска открыть:

```text
http://127.0.0.1:5000
```

Если контейнер PostgreSQL уже был создан раньше и нужно заново загрузить обновленные seed-данные:

```bash
docker compose run --rm web sh -c "pip install --no-cache-dir -r requirements.txt && python -m parsers.import_demo_data"
```

## Тесты

```bash
docker compose run --rm web sh -c "pip install --no-cache-dir -r requirements.txt && python -m pytest"
```

Ожидаемый результат:

```text
10 passed
```

## Демо-сценарий для защиты

1. Открыть dashboard.
2. Показать источник данных: PostgreSQL.
3. Выбрать специальность, например `Backend Python`.
4. Показать метрики вакансий, резюме и конкурентности.
5. Отфильтровать по грейду `Junior` или `Middle`.
6. Отфильтровать по навыку, например `Python`, `React`, `SQL` или `Kubernetes`.
7. Показать графики распределения и таблицу топ навыков.
8. Выбрать `QA Engineer`, чтобы показать fallback без вакансий.

## Ограничения MVP

- Данные подготовлены как локальная демонстрационная выборка, имитирующая результат парсинга.
- Зарплатная аналитика пока не выведена в UI, потому что такие данные часто неполные.
- Реальный внешний парсер стоит подключать после стабилизации схемы и правил обработки данных.

## Следующее развитие

- Импорт вакансий из CSV/API в таблицу `vacancies`.
- Периодический сбор агрегатов резюме без персональных данных.
- Зарплатные графики только при достаточном количестве заполненных зарплат.
- Более детальные рекомендации по навыкам для выбранного грейда.
